## 프로젝트 개발 계획서 - Alertory

### 1. 개요

- **프로젝트명**: 실시간 음성 인식 이상상황 감지 시스템(Alertory)
- **훈련과정명(소속)**: KDT 풀스택 개발자 취업캠프 (중앙정보기술인재개발원)
- **작성자(역할)**: **최정현** – 프론트/백엔드 개발, 실시간 마이크 상태 및 AI 작동 유무 시각화, UX 개선

#### 1-1. 개발 배경 및 필요성

현대 산업 현장에서는 기계 소음이 큰 환경에서 직원의 안전을 실시간으로 확인하기 어렵습니다.  
특히 공장·생산 시설에서 발생하는 사고나 위험 상황을 제때 감지하지 못하면 인명 피해로 이어질 수 있습니다.  
기존 안전 시스템은 CCTV 중심의 모니터링이나 단순 센서 알람에 의존하고 있어, 실제로 **“사람이 위험 신호를 보냈는지”**를 바로 파악하기 어렵다는 한계가 있습니다.

이를 해결하기 위해, 공장 환경에서 발생하는 **비명, 고함, 도움 요청 음성**과 같은 긴급 신호를 실시간으로 감지·분석하여 관리자에게 즉시 알림을 제공하는 **음성 기반 안전 모니터링 시스템**을 목표로 합니다.

#### 1-2. 기존 아이디어와의 차별점

- 단순 소음 감지가 아닌 **음성 이벤트 분류 + 지속 시간 기반 이벤트 발생 로직**
- **실시간 음성 분석 + 푸시 알림**을 하나의 플랫폼에서 제공
- **사용자 권한(관리자/매니저/뷰어)별 접근 제어** 및 환경 설정 기능 제공

#### 1-3. 개발 목표

1. 산업 환경에서 발생하는 긴급 음성 이벤트의 **실시간 감지**
2. 비명, 고함, SOS 등 **위험 음성 유형 분류** 및 이벤트 발생 시 관리자에게 **즉시 알림 제공**
3. 사용자 권한·환경 설정 기능을 통해 **시스템 운영 효율성 향상**

이를 통해 기존 공장 안전 모니터링 시스템이 갖고 있는  
“사고 전후의 위험 음성을 실시간으로 감지하지 못하는 문제”를 보완하는 것이 최종 목표입니다.

#### 1-4. 예상 산출물

1. 개발 계획서 및 요구사항 분석서  
2. 시스템 설계서 (아키텍처, DB 설계, API 설계)  
3. 데이터셋 구축 자료 및 전처리 코드  
4. 웹 기반 실시간 음성 모니터링 시스템 (Frontend + Backend + AI Server)  
5. AI 모델 학습 및 추론 파이프라인  
6. 테스트 보고서 및 QA/QC 자료  
7. 발표 자료 및 데모 영상

### 2. 주요 기능

- **음성 감지**
  - 실시간 마이크 입력 기반 음성 감지
  - 초기에는 **음성 파일 업로드 방식**으로 테스트 후, 단계적으로 실시간 스트리밍 감지로 전환

- **음성 분석 처리**
  - 음성 특징(피치, 에너지 등)을 활용한 비명·고함 감지
  - 향후 확장: 나이·성별 등 추가 특성 분석, 파형 기반 CNN/Transformer 적용

- **음성 이벤트 관리**
  - 특정 키워드, 비명, 고함 등에 대한 **Threshold** 설정
  - 일정 시간 이상 지속될 경우 **이벤트 발생**으로 판정
  - 발생 이벤트를 DB에 저장하고 이력 조회 지원

- **푸시 알림**
  - 관리자에게 즉시 알림 전송
  - 알림 내용: 발생 시간, 위치, 이벤트 유형, 신뢰도
  - 응급 상황 단계별 안내(예: 119 연락 가이드) 제공

- **사용자 레벨 관리 및 환경 설정**
  - 권한: 관리자(Admin), 매니저(Manager), 뷰어(Viewer)
  - 감지 민감도, 알림 ON/OFF, 운영 시간대 등 환경 설정
  - 날짜/유형/위치 필터 기반 이벤트 기록 조회 및 음성 샘플 재생

### 4. 시스템 구조 및 기술 스택

| 구분 | 기술 스택 | 주요 역할 및 핵심 기능 |
|------|-----------|------------------------|
| **Frontend** | React, Vite, Axios | SPA 대시보드, 실시간 이벤트 알림, 오디오 파일 검증 및 업로드(WAV/MP3, 10MB 이하) |
| **Backend** | Spring Boot, Spring Security, Spring Data JPA | REST API, 비즈니스 로직 제어, 세션/권한 관리, AI 서버와 WebClient 통신 |
| **AI Server** | FastAPI, PyTorch | DCCRN 기반 노이즈 제거, AST(Transformer 계열) 기반 비명·위험 음성 분류 및 추론 |
| **Database** | H2(개발용), MySQL(운영 타깃) | RBAC(역할 기반 접근 제어), 사용자 상태 관리, 이벤트 로그/신뢰도/파일 경로/시간 저장 |
| **Infra** | Ubuntu, REST API, CORS | 서비스별 포트 분리, 독립 배포, CPU 기반 추론 최적화, CORS 설정 |

상세 아키텍처와 포트 매핑은 `README.md`의 시스템 구조 및 실행 방법을 참고합니다.

### 5. 역할 및 기여 (최정현)

- **프론트엔드**
  - React + Vite 기반 대시보드 화면 설계 및 구현
  - 로그인/인증 흐름 및 사용자 상태(PENDING/ACTIVE 등) UI 구성
  - 실시간 감지 현황, 이벤트 목록, 통계/분석, 알림·관리자 페이지 등 주요 화면 제작

- **백엔드 연동**
  - Spring Boot 백엔드와의 API 연동(이벤트 조회, 업로드 결과 조회 등)
  - 세션/쿠키 기반 인증 흐름에 맞춘 프론트엔드 측 상태 관리

- **AI 서버 연동 및 UX 개선**
  - FastAPI 기반 AI 서버와의 HTTP 통신(오디오 업로드 → 분석 결과 수신) 흐름 설계
  - 업로드 가능한 오디오 형식/용량 검증, 오류 메시지 처리 등 사용자 경험(UX) 개선
  - 전체 시스템 구조를 사용자가 이해하기 쉽게 보여주는 화면/가이드 문구 정리

### 6. 기대효과 및 활용 분야

#### 기대효과

- **안전 공백 최소화**: 기계 소음이 심한 프레스/사출 공장에서 비명·구조 요청을 실시간 감지하여 인명 피해를 예방  
- **높은 판별 정확도**: 다단계 필터링과 AI 모델을 통해 외부 소음 간섭을 줄이고, 오탐·미탐을 최소화  
- **관리 효율성 향상**: 관리자에게 즉시 푸시 알림을 전송하여 빠른 초동 조치 가능, 이력 기반 사후 분석 용이

#### 활용 분야

- **스마트 안전장비**: 클린사업장 조성지원사업 등 중소사업장 스마트 안전 시스템으로 도입 가능  
- **공공 안전**: 1인 근무지, 외곽 작업장 등 사고 조기 발견이 어려운 환경에서의 안전 모니터링  
- **맞춤형 안내 서비스**: 외국인 근로자 비율이 높은 공장에서, 국적·언어별 응급 상황 안내 가이드와 연계 가능
